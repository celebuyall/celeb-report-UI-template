# PRD. UI Component Spec (Repository-Aligned)

## 목적
- 이 문서에 포함된 UI 코드 스냅샷(`App.tsx`, `data.ts`, `components/`)을 그대로 복제한다.
- 모든 텍스트/수치/아이콘은 `data.ts`에서만 가져온다.
- 데이터가 바뀌어도 레이아웃과 스타일은 동일하게 유지된다.

## 프롬프트 사용 규칙 (중요)
- 역할: UI 코드 생성기. 이 문서의 코드 스냅샷과 동일한 UI를 만드는 코드만 출력한다.
- 입력: 이 문서 + 데이터(Report로부터 생성된 결과).
- 출력: **파일별 코드만** 출력한다. 설명/요약/해설 금지.
- 실패 조건: JSX 구조/클래스 변경, 섹션 순서 변경, 파일 누락 시 실패.

## 출력 파일 목록
```
├── App.tsx              ← 메인 UI (data.ts 임포트)
├── data.ts              ← 데이터만 (01문서로 추출)
├── components/
│   ├── Icons.tsx
│   └── Logo.tsx
├── types.ts
├── index.tsx
├── index.html
├── package.json
├── tsconfig.json
└── vite.config.ts
```

## 공통 규칙
- `lucide-react` 아이콘만 사용한다. 아이콘 목록은 `components/Icons.tsx` 참조.
- `theme` 값(`purple` | `blue` | `pink`)으로 Tailwind 클래스 매핑을 통일한다.
- 섹션 제목/고정 라벨은 코드에 하드코딩 유지한다. 데이터로 대체하지 않는다.
- Recharts 차트(RadarChart, RadialBarChart)는 PRD 코드 그대로 사용한다.

## UI 동일성 규칙 (절대 준수)
- 이 문서의 JSX 구조, `className`, 그리드/간격, 섹션 순서를 1:1로 유지한다.
- 데이터 바인딩은 텍스트 노드/수치/링크/아이콘만 치환한다. 래퍼 `div` 추가/삭제 금지.
- PRD 코드 블록과 다른 지시가 불일치하면 **PRD 코드 블록이 정답**이다.

## 금지 사항
- 신규 컴포넌트/라이브러리 추가 금지.
- 파일 구조 변경 금지.
- 포맷터 적용으로 들여쓰기/줄바꿈 변경 금지.
- 데이터 스키마 구조 변경 금지.

## 데이터 바인딩 맵 (요약)

### data.ts (01문서로 추출)
- `celebData`: 셀럽 프로필 (name, handle, platform, profileUrl, category, tagline, identity[], dna[])
- `matchScore`: 매칭 점수 (score, summary)
- `productDefinition`: 제품 정의 (headline, highlight, description, quote, keyConcepts[])
- `matchPoints[]`: 매칭 포인트 (feature, connection, title, logic, isCore)
- `contentIdeas[]`: 콘텐츠 아이디어 (title, subTitle, tags[], flow[], rationale, synergy, theme)
- `strategicPillars[]`: 전략적 기둥 (tabTitle, icon, title, subtitle, context, bridge, solution, theme)
- `externalLinks[]`: 외부 링크 (label, url, icon, color)

### 섹션별 데이터 매핑
1. **CELEB PROFILE HERO**: `celebData.*`
2. **MATCHING SCORE & KEY POINTS**: `matchScore`, `matchPoints[]`
3. **PRODUCT DEFINITION (Deep Dive)**: `productDefinition`
4. **STRATEGIC FIT**: `celebData.name`, `strategicPillars[]`
5. **CONTENT STRATEGY**: `contentIdeas[]`

  const colorClasses: Record<string, string> = {
    purple: 'text-purple-700 font-semibold',
    blue: 'text-blue-700 font-semibold',
    pink: 'text-pink-700 font-semibold',
    slate: 'text-slate-900 font-semibold',
    white: 'text-white font-bold underline underline-offset-2'
  };

  return parts.map((part, i) => {
    if (part.startsWith("'") && part.endsWith("'")) {
      const keyword = part.slice(1, -1);
      return <strong key={i} className={colorClasses[color] || colorClasses.purple}>{keyword}</strong>;

## 사용 방법

### 새 프로젝트 생성 시:
1. 모든 파일을 복사
2. `npm install` 실행
3. `data.ts` 파일 교체 (01문서로 추출한 데이터)
4. `npm run dev` 실행

### 주의사항:
- **App.tsx는 절대 수정하지 마세요** - UI 코드가 고정되어 있음
- **data.ts만 교체** - 01문서로 추출한 데이터 파일로 교체
- Logo 텍스트를 변경하려면 `components/Logo.tsx` 수정

### 데이터 유동성:
- `celebData.category`: "일반"(파란) | "뷰티"(보라) | "푸드"(초록) 중 1개
- `matchPoints[]`: 핵심 소구점(isCore: true)은 3개 고정, 서브 소구점(isCore: false)은 개수 유동적
- `contentIdeas[]`: N개 유동적 (theme은 purple→blue→pink 순환 배정)
- `strategicPillars[]`: 정확히 3개 필요
- `externalLinks[]`: 개수 유동적 (0~N개)
