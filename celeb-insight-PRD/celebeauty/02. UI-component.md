# PRD 02. UI Component Spec (Repository-Aligned)

## 목적
- 이 문서에 포함된 UI 코드 스냅샷(`App.tsx`, `components/FandomDashboard.tsx`, `components/ContentDashboard.tsx`)을 그대로 복제한다.
- 모든 텍스트/수치/아이콘은 `/data.ts`의 `dashboardData`에서만 가져온다.
- 데이터가 바뀌어도 레이아웃과 스타일은 동일하게 유지된다.

## 프롬프트 사용 규칙 (중요)
- 역할: UI 코드 생성기. 이 문서의 코드 스냅샷과 동일한 UI를 만드는 코드만 출력한다.
- 입력: 이 문서 + `data.ts` 스키마/데이터(Report A/B로부터 생성된 결과).
- 출력: **파일별 코드만** 출력한다. 설명/요약/해설 금지.
- 실패 조건: JSX 구조/클래스 변경, 섹션 순서 변경, 파일 누락 시 실패.

## 공통 규칙
- 모든 강조는 `**텍스트**`로 입력하고 UI에서 강조 스타일로 파싱한다. (구체적 렌더링은 아래 규칙 참고)
- `icon` 문자열은 `lucide-react` 컴포넌트명과 매칭한다.
- `tone` 값으로 Tailwind 클래스 매핑을 통일한다. (tone ↔ 클래스 맵은 컴포넌트 상단에 정의)
- 섹션 제목/고정 라벨(예: 섹션 헤더, 탭 텍스트)은 코드에 하드코딩 유지한다. 데이터로 대체하지 않는다.
- 프로필 플랫폼 아이콘은 `profile.platform` 값으로 결정한다: `Instagram` → `Instagram` 아이콘, `YouTube` → `Youtube` 아이콘, 그 외는 `ExternalLink`.

## UI 동일성 규칙 (절대 준수)
- 이 문서의 JSX 구조, `className`, 그리드/간격, 섹션 순서를 1:1로 유지한다.
- 데이터 바인딩은 텍스트 노드/수치/링크/아이콘만 치환한다. 래퍼 `div` 추가/삭제 금지.
- 강조 처리(`**`)는 섹션별 기존 스타일을 그대로 적용한다. SWOT 섹션은 강점 카드만 보라색 강조를 사용한다.
- PRD 코드 블록과 다른 지시가 불일치하면 **PRD 코드 블록이 정답**이다.

## 금지 사항
- 신규 컴포넌트/라이브러리 추가 금지.
- 파일 구조 변경 금지.
- 포맷터 적용으로 들여쓰기/줄바꿈 변경 금지.

## 강조 렌더링 규칙 (섹션별)
- 라이트 배경 섹션은 `renderTextWithHighlight`를 사용한다. (예: Core Definition, Evidence, KPI 요약, Requirements, Strategies 설명)
- 다크 배경 섹션은 `renderTextWithBold`를 사용한다. (예: Hero, 멘트, Strategic Vision, Executive Summary)
- SWOT 섹션(전략적 제언)은 **Opportunity 카드만** `renderTextWithHighlightPurple`, Risk 카드는 `renderTextWithHighlight`를 사용한다.
- 강조 처리는 코드 블록에 정의된 함수/클래스 그대로 사용하고, 새로운 스타일 변형을 추가하지 않는다.

## 데이터 바인딩 맵 (요약)
### App.tsx
- 탭 토글은 고정 텍스트 유지, `dashboardData.meta.lastUpdated`만 날짜 영역에 바인딩.

### FandomDashboard
- Profile Header: `fandom.profile.*`
- Commerce DNA 멘트/카드: `fandom.profile.commerceDNA.subtitleLines`, `fandom.profile.commerceDNA.cards`
- KPI: `fandom.kpi.*`
- Core Definition: `fandom.coreDefinition.*`
- Evidence: `fandom.evidence[]`
- Deep Dive Top3: `fandom.deepDive[]`
- Funnel: `fandom.funnel.*`
- Buying Triggers: `fandom.buyingTriggers.*`
- SWOT: `fandom.swot.*`

## 8) App.tsx
```tsx
import React, { useEffect, useState } from 'react';
import { Users, FileText, Calendar } from 'lucide-react';
import { TabView } from './types';
import { dashboardData } from './data';
import FandomDashboard from './components/FandomDashboard';
import ContentDashboard from './components/ContentDashboard';

const App: React.FC = () => {
  const [activeTab, setActiveTab] = useState<TabView>(TabView.FANDOM);
  const [currentDate, setCurrentDate] = useState<string>('');

  useEffect(() => {
    const today = new Date();
    const formatted = `${today.getFullYear()}. ${String(today.getMonth() + 1).padStart(2, '0')}. ${String(today.getDate()).padStart(2, '0')}`;
    setCurrentDate(formatted);
  }, []);

  const lastUpdated = dashboardData.meta.lastUpdated || currentDate;

  return (
    <div className="min-h-screen bg-[#F3F5F9] text-slate-800 pb-20">
      <header className="sticky top-0 z-50 bg-white/80 backdrop-blur-md border-b border-slate-200 shadow-sm">
        <div className="max-w-7xl mx-auto px-4 sm:px-6 lg:px-8">
          <div className="flex items-center justify-between h-16 relative">
            <div className="flex items-center z-10">
              <span className="font-logo text-4xl font-normal tracking-tight text-brand cursor-pointer select-none">
                Celebeauty
              </span>
            </div>

            <div className="absolute left-1/2 top-1/2 transform -translate-x-1/2 -translate-y-1/2">
              <div className="flex bg-slate-100 p-1 rounded-lg">
                <button
                  onClick={() => setActiveTab(TabView.FANDOM)}
                  className={`flex items-center gap-2 px-4 py-2 rounded-md text-sm font-medium transition-all duration-200 ${
                    activeTab === TabView.FANDOM
                      ? 'bg-white text-brand shadow-sm'
                      : 'text-slate-500 hover:text-slate-700'
                  }`}
                >
                  <Users size={16} />
                  팬덤 지표 분석
                </button>
                <button
                  onClick={() => setActiveTab(TabView.CONTENT)}
                  className={`flex items-center gap-2 px-4 py-2 rounded-md text-sm font-medium transition-all duration-200 ${
                    activeTab === TabView.CONTENT
                      ? 'bg-white text-brand shadow-sm'
                      : 'text-slate-500 hover:text-slate-700'
                  }`}
                >
                  <FileText size={16} />
                  채널 핵심 콘텐츠 분석
                </button>
              </div>
            </div>

            <div className="hidden md:flex items-center gap-4 text-xs text-slate-400 font-medium z-10">
              <div className="text-right">
                <p className="uppercase tracking-wider">Last Updated</p>
                <div className="flex items-center justify-end gap-1 text-slate-600">
                  <Calendar size={12} />
                  <span>{lastUpdated}</span>
                </div>
              </div>
            </div>
          </div>
        </div>
      </header>

      <main className="max-w-7xl mx-auto px-4 sm:px-6 lg:px-8 py-8 animate-fade-in">
        {activeTab === TabView.FANDOM ? (
          <FandomDashboard data={dashboardData.fandom} />
        ) : (
          <ContentDashboard data={dashboardData.content} />
        )}
      </main>
    </div>
  );
};

export default App;
```

## 데이터 바인딩 체크리스트
- 모든 텍스트/수치/아이콘은 `dashboardData`에서만 읽는다.
- `**` 강조는 파싱하여 하이라이트 스타일로 렌더링한다.
- `tone`은 색상 클래스로 매핑하고, UI 클래스 구조는 현재 파일과 동일하게 유지한다.
- 고정 개수 배열은 반드시 개수 유지 (PRD/01 기준).

## 최종 출력 규칙 (Prompt)
- 출력은 **파일 경로 + 코드 블록**만 허용한다. 그 외 문장/목록/해설 금지.
- 파일 경로를 제목으로 쓰고 바로 코드 블록을 이어서 출력한다. (예: `App.tsx`, `components/FandomDashboard.tsx`)
- PRD 코드 블록과 불일치하는 부분이 있으면 PRD 기준으로 수정한다.
